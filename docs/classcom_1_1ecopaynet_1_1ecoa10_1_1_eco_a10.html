static

Retrieve accepted transactions of the commerce of the device.

Retrieve through the API the accepted transactions of the commerce configured on the current connected device. This also includes transactions from other devices of the same commerce.

The list received is partial and is limited to 10 transactions per request.

The first time the maxDate parameter can be null to retrieve the last transactions. When processing this and subsequent responses, keep track of the lowest transaction date received. This transaction date should be passed as the value of the maxDate parameter for the next request, which will only return transactions with date's lower than or equal to the value of the maxDate parameter. Note that since the maxDate parameter is inclusive, the transaction with the matching date will actually be returned again.

Parameters
maxDateMaximum transaction date of the requested transactions
Returns
A list of transactions retrieved

Initialize the EcoA10 library.

Initializes the EcoA10 library asynchronously. If the status of the library is Status#UNINITIALIZED and the passed parameters are correct, proceeds with the initialization process. If library can't proceed with initialization, would return false and no initialization would be made. If the status and the parameters are correctly passed, the library returns true immediatly and starts an asynchornous initialization process. In this process the library connects with the specified device and make the initialization process. This process includes any update that the device or the application needs to work correctly.

If you need to change the device to work with without closing your application, you need to call EcoA10#terminate() first and return to call this function.

This process can last from 10 seconds to 1 minute, depending on if it is necessary to update the terminal. We recommend to call this function in background when your application starts.

This method fires the Events.Log events.

Parameters
deviceDevice to work with
contextContext of the application
eventListenerEvents.Initialization events listener
Returns
True if the passed parameters are correct and can process to initialize library. If not, returns false.

Perform a refund transaction.

Performs a refund transaction asynchronously. If the status of the library is Status#READY and the passed parameters are correct, proceeds to perform the refund transaction process. If library can't proceed with the transaction, would return false and no transaction would be performed. If the status and the parameters are correctly passed, the library returns true immediately and starts an asynchronous refund transaction process. In this process the library connects with the device specified in the initialization process and reads the card and pin of the cardholder, to later connect with the remote host to perform the refund transaction.

This method fires the Events.Transaction events. In this events the transaction result would be received, whether it is approved or denied.

Parameters
amountThe amount to be refunded (complete or partial)
operationNumberThe operation number of the sale to be refunded
authorizationCodeThe authorization code of the sale to be refunded
transactionDateThe date of the sale to be refunded
eventListenerEvents.Transaction events listener
extraParametersA list of extra parameters to perform transaction. See ExtraParameters
Returns
True if the passed parameters are correct and the refund transaction can be performed. If not, returns false.
